# FSRS 前端设计说明

## FSRS 是什么

FSRS（Free Spaced Repetition Scheduler）是一个记忆调度算法。每次用户答完一道题，FSRS 根据答题质量（完全不会 / 勉强答对 / 正常答对 / 秒答）算出这道题什么时候会被遗忘，然后在遗忘前自动安排复习。

**核心价值：** 每个人、每道题的遗忘时间不同。FSRS 通过学习每个用户的答题历史，自动适配个性化的复习节奏——简单的题隔很久才复习，困难题频繁出现。比固定的"每天 5 题"精确很多。

## 前端需要实现的页面

### 1. 答题页（词汇 / 语法练习）

**交互流程：**

```
展示题目 → 用户选择/填写答案 → 显示对/错 
    → 展示四个评分按钮 [重来] [困难] [良好] [简单]
    → 用户点击其中一个 → 自动进入下一题
```

**四个评分的含义：**

| 按钮 | 含义 | 什么时候点 |
|------|------|----------|
| 重来 | 答错了，完全不会 | 看了答案才发现自己不知道 |
| 困难 | 答对了但很勉强 | 想了很久、不太确定、蒙对的 |
| 良好 | 正常答对 | 回忆了一下就答上来了 |
| 简单 | 秒答 | 一看就知道，不需要想 |

**设计要点：**

- 四个按钮并排排列，默认选中"良好"
- 按钮颜色做区分（如：重来=红，困难=橙，良好=绿，简单=蓝）
- 不要把这四个按钮放在同一行文字里，要足够醒目
- 桌面端可以做键盘快捷键（1/2/3/4 对应四个按钮）

**传给后端的参数：**

```json
{
  "question_id": 123,
  "rating": 3
}
```

rating 取值：1=重来，2=困难，3=良好，4=简单。

### 2. 学习计划页（今日任务 + 复习进度）

**页面元素：**

- **今日待复习数量**（醒目的大数字 + 环形进度条，如"20 题中已完成 8 题"）
- **任务列表**：每行显示题目摘要（如单词或语法点名称）、上次复习日期
- **"开始复习"按钮**：点击进入答题页
- **近 7 天复习量柱状图**（可选，简单展示即可）

**后端返回的数据结构：**

```json
{
  "today_total": 20,
  "today_done": 8,
  "tasks": [
    {
      "id": 123,
      "content": "abandon",
      "type": "vocabulary",
      "last_review": "2026-07-03"
    },
    {
      "id": 456,
      "content": "现在完成时 vs 一般过去时",
      "type": "grammar",
      "last_review": "2026-07-02"
    }
  ],
  "weekly_completed": [3, 12, 8, 15, 0, 5, 8]
}
```

### 3. 学习统计仪表盘

**页面元素：**

- **当前知识留存率**（大百分比数字 + 进度条）：所有学习过的题目中，当前还能回忆起来的比例（FSRS 算法给出）
- **已掌握题目数 / 有遗忘风险题目数 / 今日待复习数**（三列卡牌）
- **连续打卡天数**（简单的数字 + "🔥" 图标即可）

## 前后端交互汇总

| 操作 | 前端发送 | 后端返回 | 前端展示 |
|------|---------|---------|---------|
| 加载今日任务 | GET /api/tasks/today | 任务列表（见上方 JSON） | 任务列表页 |
| 提交答题评分 | POST /api/review { question_id, rating } | 状态确认 | 自动跳下一题 |
| 加载统计数据 | GET /api/stats | 留存率、掌握数、打卡天数 | 仪表盘 |
| 设定学习目标 | POST /api/user/goal { daily_minutes, target_level } | 状态确认 | 设置页 |

---

## 不需要前端做的事

- **不需要理解 FSRS 算法细节**——所有计算都在后端
- **不需要前端决定哪天复习哪道题**——复习排期完全由后端 FSRS 决定，前端只是"展示今日任务列表"和"传给后端评分"
- **不需要处理离线场景**——第一版不考虑离线缓存，网络断了就提示刷新

---

## 总结

前端新增的工作量主要是：

1. **答题页多四个按钮**（评分反馈）
2. **任务列表页**（从后端拿数据展示，约等于一个列表组件）
3. **统计仪表盘**（几个卡牌 + 柱状图，UI 库随便找一个就行）
